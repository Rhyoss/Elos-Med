## ACHADOS

### #1 🔴 (corrigido durante a auditoria) — `/suprimentos` retornava 500
**Causa**: `@dermaos/ui` não exportava `Dialog`, `Sheet`, `Label` (apenas `DialogRoot`, `SheetRoot`, e `label.tsx` ainda fora do barrel). Os 7 modals de suprimentos importavam os nomes ausentes.

**Fix aplicado** em `packages/ui/src/primitives/index.ts`:
```ts
export { DialogRoot as Dialog } from './dialog';
export * from './label';
export { SheetRoot as Sheet } from './sheet';
```

**Status**: `/suprimentos` agora carrega 200 (POSIÇÃO DE ESTOQUE / LOTES / COMPRAS / RECEBIMENTO / KITS / RASTREABILIDADE).

### #2 🔴 — `/comunicacoes` layout colapsado
**Sintoma**: Os painéis (CANAIS, lista de conversas, thread, contato) aparecem **empilhados verticalmente** em vez de em colunas. CANAIS sobe para o topo direito; thread fica preto/vazio.

**Causa provável**: O grid CSS de 4 colunas do Comunicações foi quebrado durante a migração DS — o container deixou de ter `grid-template-columns` ou `flex-direction: row`. Cada filho (`<aside>`, `<section>`, `<section>`, `<div>`) renderiza com 1336px ocupando linha inteira.

**Caminho**: `apps/web/src/app/(dashboard)/comunicacoes/page.tsx` + `_components/`.

### #3 🟠 — Sidebar trunca labels
**Sintoma**: Sidebar mostra "Dashboar", "Paciente", "Comunic.", "Suprim.", "Financei", "Analytic", "Config." com texto cortado.

**Causa**: `Sidebar.tsx` tem `width: 88px` fixo (rail vertical). Labels de até ~10 chars não cabem; truncam visualmente sem ellipsis.

**Caminho**: `packages/ui/src/ds/layouts/Sidebar.tsx`.

**Opções**:
- A) Ampliar rail para ~120-140px e mostrar labels completos
- B) Manter 88px e usar labels mais curtos (4-7 chars: "Início", "Pac.", "Comm.")
- C) Icon-only com tooltip on hover (rail de 64px)

### #4 🟠 — Lista de pacientes mostra cipher AES-GCM cru
**Sintoma**: Primeiro paciente aparece com nome `ApkN6DlNJPtMwHRl:4znDsnNuYyVgu4SnGFH5bA:jCOFzRGvZ54cq1Y5YQ` (formato `IV:ciphertext:tag` do AES-256-GCM).

**Confirmado via API**: `GET /api/trpc/patients.list` retorna o campo `name` literalmente como ciphertext para este registro. Outros pacientes (Beatriz Lima, Camila Souza...) retornam o nome em claro.

**Causa provável**: Um seed específico ou registro anonimizado pelo job LGPD ficou com o nome em formato encriptado, e o `patients.service` lê do DB sem chamar o `EncryptionService.decryptIfEncrypted()`. Indica que a layer de decryption não é aplicada em todas as queries de listagem.

**Caminhos**:
- `apps/api/src/modules/patients/patients.service.ts` (queries de lista)
- `apps/api/src/lib/encryption.ts` ou equivalente
- Verificar se o job LGPD anonymize está corrompendo nomes em vez de substituí-los

**Risco de segurança**: vazamento de formato interno (IV/tag) a usuários — não é PII, mas expõe esquema criptográfico.

### #5 🟠 — Click em linha de paciente não navega
**Sintoma**: As `<tr>` da tabela de pacientes têm `cursor: pointer` mas o handler não dispara navegação. Apenas o ícone seta `→` no fim da linha (sem href) é visível, mas não responde.

**Caminho**: `apps/web/src/app/(dashboard)/pacientes/page.tsx` — handler de `onClick={() => router.push(...)}` ausente ou erroneamente mapeado.

**Verificado**: Direct URL navigation `/pacientes/{uuid}/perfil` funciona — só o click handler está quebrado.
